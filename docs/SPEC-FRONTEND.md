# 前端代码规范

## 组件规范
模块和样式需要一一对应，模块中使用了样式的位置必须注释说明，并标注样式的具体作用
css中必须对每个样式进行注释说明

## 文档要求
遵循“代码即文档”的要求：
1. 每个模块需要有模块级文档注释，说明以下内容：
    - 模块的功能用途
    - 模块依赖的其他模块
    - 模块的使用示例
    - 模块导出的主要方法、接口、类的说明
2. 每个函数需要有函数级文档注释，说明以下内容：
    - 函数的功能用途
    - 函数的参数说明
    - 函数的返回值说明
    - 函数的异常情况说明
3. 对于每个结构体，类，需要有结构体/类级文档注释，说明以下内容：
    - 结构体/类的功能用途
    - 结构体/类的属性说明
    - 结构体/类的方法说明
    - 每个字段需要有字段级文档注释，说明字段的用途

对于不导出的私有函数、类、结构体，可以简化文档注释，但仍需说明其功能用途。

## 模块设计
1. 当模块导出方法过多时，需要评估是否是模块责任过重
2. 当模块依赖过多其他模块时，需要评估是否是模块耦合过高、责任过重
3. 模块之间的依赖应该清晰可梳理
